    • ForEach --

      When You want to do some thing with the each elemnt of an array but you don't want to get a new array but just only to show the result. Then you can use forEach. Remembar forEach function won't return anything but you can do console log it.

    • Filter --

      Filter is also a method of an array. If you want to make a new array but dependents on any condition then you must have to filter method. Filter also returns new array but depending on any condition.

  2. What is the difference between LocalStorage, SessionStorage and Cookies?

    Both of them has some own method to set data and get data. like getItems(), setItems(), clear() etc. And you will be just allowed to store string data in this store.
    • LocalStorage --

      Local storage will be available just for the same window of this browser. But Local Storage is permanent. If you don't delete it never it will be deleted. But it won't go to server side. It allows 10MB of data to be stored.

    • SessionStorage --

      SessionStorage also will be available just for the same windo of this borwser with that SessionStorage will run just for the session. Means when you will close the windo(tab) the data what is stored in SessionStorage that will be removed. So after closing the window you can't get the data again. It also won't go to server side. It allows 5MB of data to be stored.

    • Cookies --

      The storage capacity is limited to 4KB of data.The data is sent with the request from the client to the server. If you want to set expire time to remove it you are free to do it.

  3. What is the Event Loop?

    We now JS is the singleThread Programing language. It's means JS execute code line by line. But If you want to do somethin at same time then you can't do it when your programing language is singleThread. For that Event loop helpes to JS to work as Multithread language. One concept called CallStack. Call stack is a LIFO(Last in First Out) data structure. It means which function will call at the last time this function will execute first time. When you call many function one after one. Function will wait to complete it's above function. After completing the above function then this function will be executed.

  • What is the purpose of chooseQuiz function?

    যখন displayQuizOptions ফাংশন থেকে chooseQuiz ফাংশন কে onclick এর মাধ্যমে কল করা হচ্ছে তখন দুটি আরগুমেন্ট পাস করা হচ্চে পরবর্তীতে যা chooseQuiz এ প্যারামিটার হিসেবে রিসিভ হচ্ছে। প্যারামিটার এর প্রথম প্যারামিটার টি ইনডেক্স হিসেবে পরবর্তীতে ব্যবহার করা হয়েছে। এবং দ্বিতীয়টি বাছাইকৃত উত্তর হিসেবে। এরপর answers নামের অ্যারের মধ্যে ফাইন্ড মেথড ব্যবহার করে চেক করা হয়েছে যে, যে নাম্বার প্রশ্নের অপশন সিলেক্ট করা হয়েছে সেটা আগে থেকে সিলিক্ট করা আছে কিনা। যদি থাকে তবে আগের উত্তর টি splice এর মাধ্যমে রিমুভ করে নতুন জবাব এড করা হয়েছে। আর যদি আগে থেকে এই প্রশ্নের জবাব দেওয়া না থাকে তবে answers নামের অ্যারের মধ্যে পুশ পুরো অ্যারেটাকে কপি করা হচ্ছে যাতে এবং এর সাথে সিলেক্ট করা উত্তরটিও। পুশ এর মাধ্যমে কপি করার সময় spread ব্যবাহার করা হয়েছে যাতে রেফারেন্স কপি না হয়ে যা। সর্বশেষ displayAnswers ফাংশন কে পুনারায় কল করা হয়েছে।
